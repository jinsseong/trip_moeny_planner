# Supabase trips 테이블 확인 가이드

## ✅ 이미 schema_trips.sql을 실행하셨다면

추가로 할 일은 없습니다! 이제 앱을 사용할 수 있습니다.

## 🔍 테이블이 제대로 생성되었는지 확인하는 방법

### 방법 1: Supabase 대시보드에서 확인

1. Supabase 대시보드 접속
2. 좌측 메뉴에서 **"Table Editor"** 클릭
3. 다음 테이블들이 보이는지 확인:
   - ✅ `trips` 테이블
   - ✅ `expenses` 테이블에 `trip_id` 컬럼이 있는지 확인

### 방법 2: SQL Editor에서 확인

```sql
-- trips 테이블 확인
SELECT * FROM trips;

-- expenses 테이블 구조 확인 (trip_id 컬럼이 있는지)
SELECT column_name, data_type 
FROM information_schema.columns 
WHERE table_name = 'expenses' AND column_name = 'trip_id';
```

## 🚀 이제 할 수 있는 것들

1. **여행 추가하기**
   - 관리자 페이지에서 여행을 추가할 수 있습니다 (아직 UI가 없으면 API로 직접 추가 가능)
   - 또는 메인 페이지에서 여행 선택 드롭다운이 보이면 정상입니다

2. **지출 추가하기**
   - 메인 페이지에서 지출을 추가하면 선택한 여행에 자동으로 연결됩니다

3. **날짜별 그룹핑 확인**
   - 지출 목록이 날짜별로 그룹핑되어 표시됩니다

## ❓ 여전히 오류가 발생한다면

1. **브라우저 콘솔 확인** (F12)
   - 어떤 오류 메시지가 나오는지 확인

2. **서버 로그 확인**
   - 터미널에서 서버 로그를 확인하여 오류 메시지 확인

3. **테이블 확인**
   - 위의 "방법 1" 또는 "방법 2"로 테이블이 제대로 생성되었는지 확인

## 📝 참고사항

- `trips` 테이블이 비어있어도 정상입니다 (아직 여행을 추가하지 않았기 때문)
- 메인 페이지의 여행 선택 드롭다운에 "전체" 옵션만 보이면 여행이 없는 것입니다
- 여행을 추가하려면 관리자 페이지에 여행 관리 기능을 추가하거나, API를 직접 호출해야 합니다
